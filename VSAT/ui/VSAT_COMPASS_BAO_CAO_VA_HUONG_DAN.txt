I. TỔNG QUAN DỰ ÁN
================================================================================

Tên dự án: V-SAT Compass
Mô tả: Nền tảng mô phỏng thi và hỗ trợ ôn tập V-SAT đa nền tảng (Android + Web)
        với phân tích kết quả và gợi ý lộ trình học cá nhân hóa.

Kiến trúc:
  - Backend: Spring Boot 3.2.5 (Java 17) — một backend chung cho cả Android và Web
  - Database: PostgreSQL (Neon Serverless) — 27 bảng, 20 ENUM types
  - Android: Java + XML (Native Android)
  - Web: React hoặc tương đương (triển khai sau Android)
  - Build tool: Gradle
  - Auth: JWT (access token 15 phút + refresh token 30 ngày)

4 vai trò người dùng:
  1. STUDENT (Học viên) — thi thử, luyện tập, xem kết quả, báo lỗi
  2. COLLABORATOR (Cộng tác viên) — tạo câu hỏi, gửi duyệt, import Excel
  3. CONTENT_ADMIN (Admin nội dung) — duyệt câu hỏi, tạo đề, quản lý ticket
  4. SUPER_ADMIN — toàn quyền hệ thống, quản lý user & role, audit log

MVP bắt đầu với 2 môn: Toán và Tiếng Anh.
Một app Android duy nhất, 2 mode: Student mode + Admin mode (chuyển đổi theo role).

LƯU Ý QUAN TRỌNG: Android Studio chủ yếu dùng cho Android app. Tuy nhiên vì
Android Studio dựa trên IntelliJ IDEA nên vẫn mở được project Spring Boot (Gradle).
Nếu gặp vấn đề, anh có thể dùng IntelliJ IDEA Community Edition (miễn phí) cho
phần backend.

─────────────────────────────────────────────────────────────────────────────────
BƯỚC 1: CÀI ĐẶT PREREQUISITES
─────────────────────────────────────────────────────────────────────────────────

1.1. Java 17 (JDK)
  - Tải: https://adoptium.net/temurin/releases/?version=17
  - Hoặc dùng SDKMAN: sdk install java 17.0.11-tem
  - Verify: java -version → phải hiện "17.x.x"

1.2. PostgreSQL
  CÁCH 1 — Dùng Neon (cloud, khuyến khích):
    - Đăng ký: https://neon.tech (miễn phí)
    - Tạo project mới, chọn region "Asia Southeast 1" (Singapore)
    - Copy connection string

  CÁCH 2 — Cài local:
    - Tải PostgreSQL: https://www.postgresql.org/download/
    - Tạo database: CREATE DATABASE vsat_compass;

1.3. Android Studio (đã có)
  - Đảm bảo đã cài plugin "Spring Boot" nếu muốn chạy backend từ đây
  - Hoặc dùng IntelliJ IDEA Community cho backend

─────────────────────────────────────────────────────────────────────────────────
BƯỚC 2: SETUP DATABASE
─────────────────────────────────────────────────────────────────────────────────

2.1. Nếu dùng Neon:
  - Vào Neon Dashboard → SQL Editor
  - Copy toàn bộ nội dung file "vsat_database_schema.sql"
  - Paste và chạy → sẽ tạo 20 ENUM types + 27 bảng + indexes + triggers

2.2. Nếu dùng local PostgreSQL:
  Mở terminal/cmd:
    psql -U postgres -d vsat_compass -f vsat_database_schema.sql

2.3. Verify:
  Chạy: SELECT count(*) FROM information_schema.tables WHERE table_schema = 'public';
  Kết quả phải là 27 (hoặc hơn nếu tính sequence tables).

─────────────────────────────────────────────────────────────────────────────────
BƯỚC 3: SETUP PROJECT SPRING BOOT
─────────────────────────────────────────────────────────────────────────────────

3.1. Giải nén file "vsat-compass-api.zip" vào thư mục làm việc
     Ví dụ: D:\Projects\vsat-compass-api\

3.2. Tạo file .env (hoặc set biến môi trường):
  Copy .env.example thành .env rồi sửa:

    DATABASE_URL=jdbc:postgresql://ep-xxx.ap-southeast-1.aws.neon.tech/vsat_compass?sslmode=require
    DATABASE_USERNAME=your_username
    DATABASE_PASSWORD=your_password
    JWT_SECRET=<generate bằng lệnh bên dưới>

  Generate JWT Secret (chạy trong terminal):
    - Linux/Mac: openssl rand -base64 64
    - Windows (PowerShell):
      [Convert]::ToBase64String((1..64 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])
    - Hoặc dùng online: https://generate.plus/en/base64

3.3. Mở project bằng Android Studio / IntelliJ:
  - File → Open → chọn thư mục vsat-compass-api
  - Đợi Gradle sync xong (lần đầu sẽ tải dependencies, mất 2-5 phút)

3.4. Cấu hình environment variables trong IDE:
  CÁCH 1 — Trong Run Configuration:
    - Run → Edit Configurations → VsatCompassApiApplication
    - Tab "Environment variables"
    - Thêm: DATABASE_URL=...; DATABASE_USERNAME=...; DATABASE_PASSWORD=...; JWT_SECRET=...

  CÁCH 2 — Dùng plugin EnvFile:
    - Cài plugin "EnvFile" trong IDE
    - Trong Run Configuration, tick "Enable EnvFile"
    - Chọn file .env

  CÁCH 3 — Sửa trực tiếp application.yml (KHÔNG KHUYẾN KHÍCH cho production):
    - Thay ${DATABASE_URL:...} bằng giá trị thật

VI. GIẢI THÍCH TỪNG FILE TRONG PROJECT
================================================================================

--- CONFIG ---

SecurityConfig.java
  Cấu hình Spring Security. Định nghĩa:
  - Endpoint nào public (auth/**, swagger, GET subjects, GET exams)
  - Endpoint nào cần role gì (/admin/users/** → SUPER_ADMIN, /admin/** → CONTENT_ADMIN+)
  - JWT filter chain, CORS, session stateless, password encoder (BCrypt)

OpenApiConfig.java
  Cấu hình Swagger UI: tiêu đề, mô tả API, nút Authorize cho JWT.

JpaConfig.java
  Bật @EnableJpaAuditing để tự động fill createdAt/updatedAt.

--- SECURITY ---

JwtUtils.java
  Utility tạo access token (15 phút) và refresh token (30 ngày).
  Parse token lấy userId, role. Validate token signature + expiration.

JwtAuthenticationFilter.java
  Filter chạy trước mọi request. Đọc header "Authorization: Bearer <token>",
  validate, rồi set Authentication vào SecurityContext.

CustomUserDetails.java
  Adapter giữa entity User và interface UserDetails của Spring Security.
  Map role thành GrantedAuthority: "ROLE_STUDENT", "ROLE_CONTENT_ADMIN", etc.

CustomUserDetailsService.java
  Load user từ DB theo email hoặc id, trả về CustomUserDetails.

--- ENTITY ---

User.java
  Map với bảng "users". Dùng @Enumerated(EnumType.STRING) cho role, status, gender.
  Tự set default: role=STUDENT, status=ACTIVE, emailVerified=false.

RefreshToken.java
  Map với bảng "refresh_tokens". Mỗi token liên kết 1 user (ManyToOne).

--- REPOSITORY ---

UserRepository.java
  findByEmail, existsByEmail, findByFilters (lọc theo role, status, keyword với paging).

RefreshTokenRepository.java
  findByTokenAndRevokedFalse, revokeAllByUserId, revokeByToken.

--- DTO ---

ApiResponse.java
  Wrapper chuẩn hóa response:
  - Thành công: { data: {...}, message: "Success" }
  - Lỗi: { error: { code: "...", message: "...", details: {...} } }

AuthRequest.java
  6 inner classes: Register, Login, RefreshToken, ForgotPassword, ResetPassword,
  ChangePassword, UpdateProfile. Có validation annotations (@NotBlank, @Email, @Size).

AuthResponse.java
  TokenPair (accessToken + refreshToken + user info).
  UserInfo (id, email, fullName, role, status, ...).

--- SERVICE ---

AuthService.java (interface)
  7 methods: register, login, refreshToken, logout, getMe, updateProfile, changePassword.

AuthServiceImpl.java (implementation)
  Business logic đầy đủ:
  - Register: check email trùng, hash password, tạo user, generate token pair
  - Login: verify email+password, check status (locked/deactivated), update lastLoginAt
  - RefreshToken: validate trong DB, check expiration, revoke cũ, tạo mới
  - Logout: revoke refresh token
  - ChangePassword: verify mật khẩu cũ, hash mới, revoke tất cả token (force re-login)
  - UpdateProfile: parse gender enum, parse date, update fields

--- CONTROLLER ---

AuthController.java
  7 REST endpoints, map với API spec:
  POST /auth/register      → AU-01
  POST /auth/login         → AU-02
  POST /auth/refresh       → AU-03
  POST /auth/logout        → AU-04
  GET  /auth/me            → AU-07
  PUT  /auth/me            → AU-08
  PUT  /auth/me/password   → AU-09

--- EXCEPTION ---

AppException.java
  Custom exception với HttpStatus + error code.
  Factory methods: badRequest(), unauthorized(), forbidden(), notFound(), conflict().

GlobalExceptionHandler.java
  @RestControllerAdvice bắt tất cả exception:
  - AppException → trả đúng status + error JSON
  - MethodArgumentNotValidException → trả 400 + chi tiết field nào lỗi
  - BadCredentialsException → trả 401
  - AccessDeniedException → trả 403
  - Exception → trả 500

--- UTIL ---

SecurityUtils.java
  Helper methods để dùng trong service/controller:
  getCurrentUserId(), getCurrentUserRole(), isAdmin(), isSuperAdmin().

--- MAPPER ---

UserMapper.java
  Chuyển User entity → AuthResponse.UserInfo DTO.

VIII. CÁC MODULE CẦN CODE TIẾP THEO (THỨ TỰ ƯU TIÊN)
================================================================================

Dưới đây là thứ tự nên code tiếp, sắp theo dependency và độ quan trọng:

──── MVP PRIORITY 1 — NỀN TẢNG (nên làm ngay) ────

MODULE 1: Subject & Topic (7 API — SB-01 đến SB-07)
  Lý do: Đơn giản nhất, là dữ liệu nền cho Question và Exam.
  Entity: Subject, Topic, Subtopic
  Controller: SubjectController
  Thời gian ước lượng: 2-3 giờ

MODULE 2: Question Bank (5 API MVP — QS-01 đến QS-05)
  Lý do: Là dữ liệu lõi, cần có trước khi tạo đề.
  Entity: Question, QuestionOption, QuestionVersion, QuestionGroup
  Controller: QuestionController
  Lưu ý: CTV chỉ xem/sửa câu của mình, Admin xem/sửa tất cả
  Thời gian ước lượng: 5-7 giờ

MODULE 3: Review Workflow (5 API — RV-01 đến RV-05)
  Lý do: Liên kết chặt với Question Bank.
  Entity: QuestionReview, QuestionComment
  Controller: ReviewController
  Thời gian ước lượng: 3-4 giờ

──── MVP PRIORITY 2 — EXAM ENGINE (trái tim sản phẩm) ────

MODULE 4: Exam Management (9 API — EX-01 đến EX-09)
  Lý do: Cần có đề trước khi thi.
  Entity: Exam, ExamQuestion
  Controller: ExamController (student), AdminExamController (admin)
  Thời gian ước lượng: 5-6 giờ

MODULE 5: Exam Session Engine (7 API — ES-01 đến ES-07)
  Lý do: Đây là TRÁI TIM — cho phép học viên thi thử end-to-end.
  Entity: ExamSession, SessionAnswer
  Controller: ExamSessionController
  Logic: start session → save answers → submit → grade → return result
  Thời gian ước lượng: 7-10 giờ (phức tạp nhất)

──── MVP PRIORITY 3 — BỔ TRỢ ────

MODULE 6: History & Analytics (4 API — HI-01 đến HI-04)
  Entity: UserTopicStats (cache bảng thống kê)
  Controller: StudentStatsController

X. LƯU Ý KỸ THUẬT QUAN TRỌNG
================================================================================

1. ENUM MAPPING VỚI POSTGRESQL
   PostgreSQL dùng custom ENUM type (CREATE TYPE user_role AS ENUM ...).
   Trong entity, dùng:
     @Enumerated(EnumType.STRING)
     @Column(columnDefinition = "user_role")
     private UserRole role;
   Nếu gặp lỗi type mismatch, thêm vào application.yml:
     spring.jpa.properties.hibernate.jdbc.lob.non_contextual_creation: true

2. NEON POSTGRESQL — SSL
   Connection string Neon phải có ?sslmode=require
   Ví dụ: jdbc:postgresql://ep-xxx.aws.neon.tech/vsat_compass?sslmode=require

3. HIBERNATE ddl-auto = validate
   Đã set "validate" chứ KHÔNG phải "update" hay "create".
   Schema do file SQL tạo, Hibernate chỉ validate mapping.
   Nếu entity không khớp schema sẽ báo lỗi ngay khi start — đây là ý đồ để
   phát hiện sai sót sớm.

4. JWT SECRET PHẢI ĐỦ DÀI
   jjwt 0.12.x yêu cầu key ít nhất 256-bit cho HMAC-SHA.
   Nếu secret quá ngắn sẽ báo lỗi WeakKeyException.
   Dùng: openssl rand -base64 64 để generate.

5. CONTEXT PATH
   Server chạy tại: http://localhost:8080/api/v1/
   Tất cả URL trong controller là relative: /auth/register → full: /api/v1/auth/register

6. CTV (COLLABORATOR) CHỈ XEM DỮ LIỆU CỦA MÌNH
   Khi code Question/Review module, nhớ check:
   - Nếu role = COLLABORATOR → filter theo created_by = currentUserId
   - Nếu role = CONTENT_ADMIN hoặc SUPER_ADMIN → xem tất cả

7. SOFT DELETE
   KHÔNG dùng cờ deleted. Dùng status = ARCHIVED hoặc HIDDEN.
   Giữ dữ liệu để audit trail.

8. GIT WORKFLOW GỢI Ý
   - main: code ổn định
   - develop: code đang phát triển
   - feature/auth: module auth (đã xong)
   - feature/subjects: module subject & topic
   - feature/questions: module question bank
   - feature/exam-engine: module thi thử
